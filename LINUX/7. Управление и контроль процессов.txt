Запуск интерактивного процесса и перемещение его в фоновой режим
ping 127.0.0.1 ctrl + z
В данный момент процесс “замораживается”, он не выполняется и 
не потребляет процессорное время.

job отображает приостановленные процессы
командой bg можно продолжить процесс
fg продолжит выполнение процесса и вернет его выполнение на передний план
Выполнение команды disown -a приведёт к удалению всех задач из текущей оболочки

Запуск команды ps с ключами, позволяющими увидеть процессы всех пользователей и параметром PPID 

ps axjf | less

Количество процессов, работающих из pts (какое корректное TTY у pts?):
ps -t pts | wc

Вывод процесса, который наиболее потреблял процессорное время

ps -e --sort -time | head -n 2

Количество процессов в состоянии D (процесс в полной (непрерываемой) спячке, например, ожидает ввода/вывода)

ps -e -o stat | grep D | wc

Просмотр времени запуска процессов

ps -e -o start.pid | less

a)	top -d 5 – запуск утилиты с частотой обновления отображения в 5 секунд
b)	top -u user – вывод только лишь процессов пользователя user (-u - являющегося effective)
c)	top -p 1024, 2048, 4096 – мониторинг процессов с IDs 1024, 2048, 4096
d)	top -n 10 – максимальное число итераций, которые утилита top должна провести, а затем завершиться
e)	top -b -n 12000 > somefile – запуск top в batch режиме (в этом режиме top не принимает интерактивный ввод). В этом случае обязательно указывается параметр -n, указывающий количество итераций. (Возможно опечатка? Имелось ввиду >> ?) И выводит результат в файл somefile

1)	<Shift+N> - отсортировать задачи по PID
2)	 <Shift+P> - отсортировать по использование CPU
3)	<Shift+M> - отсортировать по использованию памяти
4)	<Shift+T> - отсортировать по времени использования CPU
5)	d – установить задержку между обновлениями
6)	= - убирает ограничения вывода
7)	u – показать процессы для указанного UID
8)	n – количество задач для отображения
9)	K – убить указанную задачу (но это для k, а K – не определено)
